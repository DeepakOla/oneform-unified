### Scrapling Flow
1. Admin enters a government portal URL (e.g., `https://scholarships.gov.in/...`) in the OneForm dashboard
2. Dashboard calls `POST /api/admin/form-templates/scrape` with `{ url }`
3. OneForm API calls Scrapling: `POST http://localhost:8001/scrape?url={url}`
4. Scrapling uses **Playwright** (`DynamicFetcher`) to load the JS-heavy govt portal
5. Scrapling extracts form field CSS selectors + labels
6. If field labels are in Hindi вҶ’ Scrapling calls **Ollama** (port 11435, Qwen 2.5 model) for translation/classification
7. Returns `{ fields: { "aadhaar": "#aadhaarNo", "name": "#applicantName", ... } }`
8. Stored as `FormTemplate.fieldMappings` in **OneForm's PostgreSQL** (port 5433)

### Why This Doesn't Disturb Power
- Scrapling is a **new container** (`oneform-scrapling`, port 8001) вҖ” Power has nothing on 8001
- Ollama is **stateless** вҖ” calling it from Scrapling is the same as calling it from Power; no interference
- Scrapling writes to **OneForm's database** (port 5433), never touches Power's database (port 5432)
- Scrapling runs in its own container with its own Playwright browser вҖ” no shared browser state
