## 1. Project Overview

OneForm is an AI-powered Indian government form autofill platform. Multi-tenant SaaS targeting:
- **Citizens (CITIZEN)** — personal form filling
- **CSC/Village Operators (OPERATOR)** — Arthiyas serving farmers/villagers (core paying segment)
- **Businesses (BUSINESS)** — bulk form submissions for employees/clients

**Stack:** Node 24 + Express 5 + Prisma 6 + PostgreSQL 17 + Redis 8 + React 19 + Vite 6 + shadcn/ui + Chrome Extension MV3 + Turborepo + pnpm

## 9. Architecture Anti-Patterns to AVOID

These were bugs in v1 that have been explicitly fixed:

1. **NEVER use `localStorage.clear()` except in logout** — wipes unrelated storage keys
2. **NEVER encrypt at display time** — encrypt at save time (form submit)
3. **NEVER store plaintext PII in DB** — Section A must always go through `encryption.service.ts`
4. **NEVER use floats for monetary values** — all money is integer paisa
5. **NEVER embed role-specific routes as separate top-level paths** — use the unified `/dashboard` with role-aware Sidebar
6. **NEVER import React explicitly in TSX files** — React 19 handles this automatically
7. **NEVER use `Redis.default()` constructor pattern** — use named import `import { Redis } from 'ioredis'`
8. **NEVER use `assert { type: 'json' }` import syntax** — use `with { type: 'json' }`

## 10. Bilingual Strategy Summary

Target: Hindi as primary secondary language (alongside English), with room to add Tamil, Telugu, Marathi later.

- **Library:** `react-i18next` (industry standard, tree-shakeable)
- **Detection order:** 1) saved preference in `user.preferences.language`, 2) browser language header, 3) default to `'en'`
- **Namespace structure:** `common`, `auth`, `dashboard`, `profile`, `documents`, `wallet`
- **RTL:** Not needed for Hindi/Indian scripts (all LTR)
- **Font:** Current sans-serif stack handles Devanagari in modern browsers. Add `Noto Sans Devanagari` as fallback in `tailwind.config.ts` if Devanagari glyphs look poor.
- **Key UX principle:** English labels always visible for field names that don't have a common Hindi equivalent (e.g., "Aadhaar", "PAN", "GSTIN" stay as-is)
